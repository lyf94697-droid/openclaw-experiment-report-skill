# CSDN 参考内容如何避免照抄

<!-- project-readiness:do-not-copy -->
<!-- project-readiness:fact-source -->

CSDN、博客园、教材网页和课程讲义只能作为参考材料。写实验报告时可以参考它们的概念、流程和命令，但不要照抄原文表达，也不要把别人的运行结果冒充成自己的结果。

## 基本原则

- 事实来源和文字表达分开处理：参考文章提供背景、命令和步骤，你自己的截图、输出和数据决定实验结果。
- 保留实验真实性：结果部分只能写你提供的真实截图、命令输出、代码运行结果或明确说明的实验现象。
- 改写不是替换同义词：应按实验要求重新组织结构，用自己的实验对象、环境和结果解释过程。
- 教程代码只能标成参考实现：如果你没有提供自己的代码，不要写成“本人编写如下代码”。
- 多来源交叉时优先用户材料：老师要求、学校模板、截图证据和本地数据优先级高于公开文章。

## 可用内容

可以从参考文章提取：

- 实验背景和概念解释
- 常用命令、参数含义、操作顺序
- 可能出现的现象和排错思路
- 术语定义和章节组织灵感

不应该直接复用：

- 原文段落和固定套话
- 别人的截图、IP、学号、运行结果
- 与本次实验不一致的设备型号、软件版本、网络拓扑
- 文章里的主观评价和个人结论

## 推荐写法

把参考资料变成“约束和事实清单”，再生成报告：

```text
参考文章说明 ping 可以验证主机连通性。
本次实验真实截图显示 192.168.10.11 可以 ping 通 192.168.10.12。
报告结果部分应写本次地址和截图结论，不复述原文案例中的地址。
```

错误写法：

```text
照搬文章中的网络地址、输出结果和完整段落。
```

正确写法：

```text
根据本次拓扑配置两台主机的静态地址，并使用 ping 命令验证互通。截图结果显示两台主机之间可以正常收发 ICMP 报文，说明局域网基础连通性配置成功。
```

## prompt 约束

生成正文时建议加入这样的约束：

```text
参考资料只用于理解实验背景、操作顺序和术语。不要照抄参考文章原句；结果部分必须以我提供的截图、命令输出和数据为准；如果缺少真实结果，请标明需要补充，不要编造。
```

`scripts/prepare-report-prompt.ps1` 也会在拼接参考资料时加入英文版约束：reference materials may be used for context, but should not be copied verbatim.

## 自检清单

交付前检查：

- 报告里是否出现了参考文章中的特殊 IP、用户名、路径或截图编号
- 结果部分是否能对应到你自己的截图或命令输出
- 大段理论说明是否换成了本实验需要的简洁解释
- “问题分析”和“总结”是否回扣本次实验，而不是泛泛套话
- 若引用公开资料，是否只保留必要事实，没有长段复述

## 项目边界

这个项目能降低照抄风险，但不能自动保证学术合规。最终提交前仍应由使用者确认学校要求、引用规范和实验真实性。
