# Code of Conduct

## Our Standard

This repository is meant to be practical, respectful, and useful.

Examples of behavior that contribute to a healthy environment:

- giving technical feedback with concrete reasoning
- reporting bugs with reproducible details
- discussing tradeoffs without personal attacks
- being clear about limitations, assumptions, and uncertainty
- helping new users understand the repository's workflow

Examples of unacceptable behavior:

- harassment, insults, or discrimination
- personal attacks instead of technical criticism
- deliberately misleading bug reports or benchmarks
- publishing private information without permission
- disruptive spam, trolling, or low-effort issue flooding

## Scope

This Code of Conduct applies in repository issues, pull requests, discussions around releases, and other project communication spaces that are clearly associated with this repository.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct.

## Reporting

If you encounter unacceptable behavior, open a private security-style report through the repository maintainer's preferred contact channel when available. If that is not available, open a minimal public issue without sensitive details and ask for a private follow-up.

## Notes

This document is intentionally short. The goal is not ceremony; it is to keep collaboration technically rigorous and socially sane.
