# Common Report Structures

## 1. Default software or programming experiment

1. 实验目的
2. 实验环境
3. 实验原理或任务要求
4. 实验步骤 / 关键代码说明
5. 实验结果
6. 问题分析
7. 实验总结

## 2. Network or hardware verification experiment

1. 实验目的
2. 实验设备与环境
3. 实验原理
4. 实验步骤
5. 实验现象与结果记录
6. 结果分析
7. 总结与思考

## 3. Fast quality checklist

- 先写用户实际做了什么，再写为什么这样做。
- 结果部分写可观察到的输出、现象、截图结论或实验数据。
- 分析部分要回扣实验要求，不要只写空话。
- 总结部分控制在 1 到 2 段，避免堆砌“收获很大”式套话。
- 如果用户给了代码、报错、截图，要在结果或分析中体现它们。

## 4. Missing-data rule

- 没有真实数据时，不要冒充真实实验结果。
- 若用户允许“样例版”或“示例版”，可以补合理假设，但要标成假设或示例。
