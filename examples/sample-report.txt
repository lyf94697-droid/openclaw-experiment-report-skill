课程名称：计算机网络
实验名称：局域网搭建与常用 DOS 命令使用

实验目的

通过本次实验掌握基础局域网的搭建方法，理解同一网段内主机进行通信所需要满足的条件，并熟悉 ipconfig、ping、arp 等 DOS 命令在网络实验中的典型用途。

实验环境

实验环境为 Windows 11 主机、VMware Workstation 以及两台 Windows Server 2019 虚拟机。两台虚拟机位于同一 192.168.10.0/24 网段，地址分别为 192.168.10.11 和 192.168.10.12，子网掩码均为 255.255.255.0。

实验原理或任务要求

在同一局域网中，若主机地址位于同一网段且子网掩码配置一致，则主机之间能够直接进行通信。ipconfig 用于查看本机网络配置，ping 用于测试连通性，arp -a 用于查看邻居缓存，从而验证通信建立后的地址解析结果。

实验步骤

首先启动两台 Windows Server 2019 虚拟机并接入同一仅主机网络，然后分别配置静态 IPv4 地址。完成地址设置后，在两台主机中执行 ipconfig 检查配置结果，再通过 ping 命令进行双向互通测试，最后使用 arp -a 查看邻居缓存记录。

实验结果

实验结果表明，两台主机的网络配置正确，ipconfig 输出与地址规划一致。ping 测试能够正常收到响应且无丢包，arp -a 输出中能够看到对端主机记录，说明本次局域网搭建已经成功完成。

问题分析

如果 IP 地址或子网掩码配置错误，即使虚拟机处于同一虚拟网络环境中，也可能导致通信失败。只有将 ipconfig、ping 和 arp 的结果结合起来分析，才能更完整地判断当前局域网中的实际网络状态。

实验总结

本次实验使我进一步掌握了局域网搭建与常用 DOS 命令使用的方法，并加深了对 IP 地址、子网掩码、局域网通信与 ARP 地址解析过程的理解。
