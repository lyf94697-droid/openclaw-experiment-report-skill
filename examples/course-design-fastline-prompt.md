# Course Design Fastline Prompt

```text
/experiment-report

请直接调用本仓库的本地 PowerShell 快线生成课程设计报告 docx，不要停在正文草稿，也不要默认跑 full 验证链路。

目标：
- 材料、模板和截图路径齐全时，优先快速产出一份可检查的课程设计报告 docx
- 显式使用 course-design-report，不要污染普通实验报告主线
- 优先保持快线稳定，贴近学校课程设计模板；不保证任意模板一次跑对

报告类型：课程设计报告
ReportProfileName：course-design-report
PipelineMode：fast
DetailLevel：full
StyleProfile：auto

参考资料：
- 参考链接或本地材料：<填写课程设计任务书、教程、需求说明或项目说明>

模板路径：
- <填写课程设计模板 doc/docx 路径>

本地截图路径：
- <截图1>
- <截图2>
- <截图3>
- <截图4>

输出目录：
- <填写输出目录>

学生信息：
- 学生姓名：李亦非
- 学号：244100198
- 班级：24C
- 指导教师：李老师
- 课程名称：<课程名称>
- 课题名称：<课程设计题目>
- 设计类别：课程设计
- 完成时间：<完成时间>
- 设计地点：<设计地点，未知可留空>

正文要求：
1. 生成完整中文课程设计报告，不要写成提纲。
2. 正文必须包含：摘要、关键词、设计目标、开发环境、需求分析、方案设计与实现、实现结果、问题与改进、设计总结、参考文献。
3. 如果模板或优质示例包含评分表，要保留模板结构；没有评分表的短模板不要临时造一页评分表。
4. 方案设计与实现要写清楚模块划分、关键设计、实现过程和为什么满足目标。
5. 实现结果要结合截图内容说明，不要只写“见图”。
6. 问题与改进要写真实可能遇到的问题、原因和后续改进方向。
7. 设计总结要像学生真实完成课程设计后的总结，不要空泛套话。
8. full 模式正文目标约 2200-3200 字；材料很少时也要保证结构完整，不要用空话撑字数。
9. 需要小节编号时使用固定文本编号，例如 3.1、3.2、4.1、4.2。
10. 不要出现“以下是一份”“可根据实际情况修改”“TODO”“待补充”“示例”“AI生成”等痕迹。

截图插入要求：
1. 截图必须真实插入最终 docx，不要只把路径写进正文。
2. 实现过程截图优先放到“方案设计与实现”，运行界面和测试结果截图优先放到“运行结果”。
3. 每张图都要生成中文图注，例如“图1 课程设计实现过程截图”“图2 运行结果截图”。
4. 同一章节连续多张截图时，默认每行 2 张。
5. 并排两张截图时，可见内容要一样大，不要靠不可见白边或透明边凑齐。
6. 图片宽度和图注要适合课程设计模板，不要撑破页面。
7. 如果报告需要流程图，优先使用本仓库的黑白 PNG 流程图生成能力；流程图要尽量放大到能看清文字，但不要超过页面边界。

执行要求：
1. 优先使用 scripts/build-report-from-url.ps1。
2. 关键参数必须包含：
   - ReportProfileName course-design-report
   - PipelineMode fast
   - TemplatePath 使用上面的模板路径（.doc 模板可直接传入，脚本会在输出目录内转成 docx 后填充）
   - ImagePaths 使用上面的截图路径
   - OutputDir 使用上面的输出目录
   - StyleProfile auto
3. 默认不要跑 full validation、layout-check、image-placement-plan；只有脚本失败或最终版式明显异常时再补跑。
4. 如果模板路径、截图路径或 `.doc` 转换依赖不可用，请直接说明，不要假装已经生成成功。
5. 最终至少输出：
   - 最终 docx 路径
   - summary.json 路径
   - pipeline-trace.json 路径
   - pipeline-trace.md 路径

请直接开始执行，不要停在“我可以帮你”的说明阶段。
```
