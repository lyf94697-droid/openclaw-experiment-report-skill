# 典型案例目录

这个目录把原来散落的 prompt、metadata、requirements 和截图规格按场景组织起来。每个案例都回答三个问题：

- 适合什么实验
- 输入材料怎么准备
- 可以跑哪条命令验证

## 案例列表

| 案例 | 类型 | 适合场景 |
| --- | --- | --- |
| [network-dos](network-dos/README.md) | 普通实验报告 | 计算机网络、局域网、DOS 命令、截图证据较多 |
| [os-process-scheduling](os-process-scheduling/README.md) | 普通实验报告 | 操作系统、算法过程、运行结果和问题分析 |
| [course-design-student-management](course-design-student-management/README.md) | 课程设计报告 | 课程设计、系统需求、模块设计、运行截图 |

## 使用建议

- 想跑完整本地 demo：先用 `network-dos`
- 想看纯正文生成约束：看 `os-process-scheduling`
- 想看课程设计 profile：看 `course-design-student-management`

这些案例里的学生信息都是演示数据。正式使用时请替换成真实课程、实验名称和个人信息。
